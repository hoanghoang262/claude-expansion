# Phase Routing Decision

## Task: "build login feature"

## Routed Phase: Implementation

## Reasoning:

1. **Action-oriented verb**: "Build" directly indicates construction/implementation work
2. **Feature already identified**: "login feature" is specified — no ambiguity about what needs to be done
3. **No preliminary work indicated**: Task doesn't mention analysis, planning, or testing phases
4. **Standard development workflow fit**: Implementation is the natural next step after planning for a clearly-defined feature

## Why Not Other Phases:

- **Requirements/Analysis**: Would require ambiguous/unclear requirements
- **Planning**: Would require breaking down into subtasks first
- **Testing**: Would come after implementation is complete
- **Review**: Would come after implementation and self-testing
