# Routing Decision

## Question: "không biết nên dùng Redis hay MongoDB"

## Selected Phase: **Clarification/Requirements Gathering**

## Reasoning:

1. **Insufficient context** - The question doesn't provide:
   - Specific use case (cache? session? document storage? real-time?)
   - Type of data to store
   - Performance/scalability requirements
   - Existing tech stack

2. **Both databases are good but for different purposes:**
   - **Redis**: In-memory, ideal for cache, session, real-time, message queue
   - **MongoDB**: Document-oriented, ideal for flexible schema, complex queries

3. **Cannot make recommendation without requirements** - Answering "use Redis" or "use MongoDB" without knowing the specific use case would be premature.

## Conclusion:

Route to **Clarification Phase** - Need to gather more information about the user's specific use case before any meaningful recommendation can be made.
